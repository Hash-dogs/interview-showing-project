## 一、postgresSQL数据库+pgvector向量数据库插件
### 1、docker配置postgresSQL数据库+pgvector向量数据库
```angular2html
C:\Users\14207> docker run -d `
>>   --name my_pgvector `
>>   -p 5432:5432 `
>>   -e POSTGRES_PASSWORD=Gyb@2005412 `
>>   -v D:\docker\PostgreSQL_PGvector:/var/lib/postgresql/data `
>>   pgvector/pgvector:pg17
```

* 补充对于wsl2的docker desktop无需设置扫描目录，因为wsl2是运行docker扫描整个磁盘的。所以容器挂载目录无任何固定，但是不要和docker镜像文件放在一起即可。

### 2、为什么选择postgresSQL数据库而不是MYSQL数据库
* 由于pgvector向量数据库插件的存在，使得向量可以存储在sql数据库中，并且在查询的时候可以与参与查询（如相似度排序等）
* MYSQl没有相关对应插件，但是在处理复杂SQL查询的时候有天然优势
* 相较于其他专业的向量数据库redis stack等，插件的优势就是可以结合SQL语句一同查询，劣势高维向量查询性能远不及专业的，并且不支持多模态？


## 二、RustFS
* 注意使用端口的url访问时不能使用翻译，否则会挂载不出来！
* RustFS：对象挂载服务，对软件文件（图像，视频，文档等）镜像存储和共享，别的服务可以使用url访问获得。
### 安装
```angular2html
docker run -d `
>>   --name rustfs_local `
>>   -p 9000:9000 `
>>   -p 9001:9001 `
>>   -e RUSTFS_ACCESS_KEY=rustfsadmin `
>>   -e RUSTFS_SECRET_KEY=rustfsadmin `
>>   -e RUSTFS_CONSOLE_ENABLE=true `
>>   -v D:/docker/RustFS:/data `
>>   rustfs/rustfs:latest `
>>   --console-enable `
>>   /data
```

### 为什么使用rustfs
#### Q1：为什么用对象存储而不是存数据库：
  数据库存文件会导致：数据库体积暴增，查询变慢。数据库备份困难。无法直接生成访问 URL
#### Q2：为什么不存服务器本地磁盘：
多台服务器部署时文件无法共享。服务器宕机文件丢失。无法水平扩展
#### Q3：对象存储的核心优势：
- 文件和业务数据解耦
- 天然支持分布式，任何服务都能访问
- 可生成带时效的临时访问链接（私密文件）
- 存储容量几乎无限扩展
- 兼容 S3 协议，迁移成本极低

### 对于文件上传删除等操作离不开S3Client类，其中的requestbody是一个关键的类
* 区别于@Requestbody注解，是接受前端HTTP请求体json中的数据
* RequestBody是AWS S3 SDK中的普通类,专门用于文件上传
* 对于本地磁盘文件上传Paths.get("文件名“)会**被spring自动有文件绝对路径转换为RequestBody实体**
* 对于由MultipartFile接受的前端文件，需要使用RequestBody.from(file)或fromInputStream（file,sive）转换为RequestBody实体。fromInputStream对于大文件可以压缩上传流提升速率。


### S3客户端是什么
* S3 全称 Simple Storage Service，是亚马逊 AWS 推出的云存储服务，它定义了一套操作文件的标准接口（API 协议）。对于Rustfs和MinIO时遵守S3标准的文件对象存储服务。
