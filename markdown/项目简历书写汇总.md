# 面试版本（依据侧重点解析更改）

## 后端+AI应用与流式输出
```markdown
- 项目描述:
集成大语言模型能力的智能面试辅助平台，提供简历AI分析、模拟面试问答、知识库RAG检索三大核心功能，支持多文档上传、流式对话与报告导出。
- 技术栈:
Java 21 + Spring Boot 4.0 + Spring AI 2.0 + PostgreSQL + pgvector + Redis + Reactor + Apache Tika
+阿里DashScope + MapStruct 工作职责:
- 深度集成Spring AI 2.0框架，封装StructuredOutputInvoker统一处理结构化输出解析失败的重试逻辑，支持配置化重试次数与错误注入，提升LLM调用的可靠性。
- 实现基于PostgreSQL + pgvector的RAG检索增强系统，采用文档分块（TokenTextSplitter）+ 阿里云text-embedding-v3向量化（1024维）+ HNSW索引存储，满足中小规模知识库的检索性能需求。
- 设计基于Query Rewrite的RAG增强架构，通过LLM将短查询、模糊语义转化为可检索语句， 配合短Token物理命中判定，将语义检索准确率提升35%以上。
- ?构建RAG会话持久化管理方案，采用"预创建占位+异步流式更新"策略，确保存储层能够实时记录AI逐字生成的完整对话内容。
- 设计多级模拟面试引擎，支持主问题与追问（Follow-up）的线性问答流，通过持久化会话状态机确保面试体验的连贯性与深度。
- 基于SSE实现流式响应，设计探测窗口算法（120字符缓冲）快速识别"无信息"模板并归一化拒答话术，避免长篇无效输出，同时保持打字机效果的用户体验。
- 基于RISEN框架设计AI评估提示词，通过资深专家角色设定、量化评分准则与否定约束，确保评估结果的专业性与一致性。
- 实现问题历史去重机制，将已回答问题（限制30道）作为上下文传入AI，自动生成不重复的新问题，提升面试体验。
- 使用MapStruct实现领域实体与DTO的编译期自动映射，性能优于反射方案，结合Lombok配置增强复杂对象转换的类型安全性。
- 实现基于文件内容特征SHA-256哈希的去重检测逻辑，在解析前拦截重复文档上传，有效避免AI 接口的冗余调用与存储资源浪费。
- 实现向量化分批处理策略，针对阿里云DashScope API限制（batch_size ≤ 10），自动分批调用，向量化吞吐量达500+文档/分钟。
```


## 侧重后端的架构与设计
```markdown
项目描述:
基于大语言模型的简历分析与模拟面试系统，为求职者提供简历评估建议、个性化面试问题生成和AI 对话练习服务，支持简历/知识库文档上传、智能问答与PDF报告导出等功能。
技术栈:
Java 21 + Spring Boot 4.0 + Spring AI 2.0 + PostgreSQL + pgvector + Redis + Redisson 4.0 + Apache
Tika + iText 8 + Docker + Gradle 8.14 工作职责:
启用Java 21虚拟线程，利用轻量级线程模型大幅提升I/O密集型场景（SSE长连接、AI调用）的并发吞吐，无需调整线程池配置即可支持千级并发。
设计Redis Stream模板化异步消费架构，实现简历分析/知识库向量化/面试报告生成的异步解耦，支持自动重试与并发消费，接口响应时间从15s降低至200ms。
设计分批评估流水线，通过动态切片（默认8题/批）规避LLM上下文窗口溢出风险，引入二次总结降级机制确保评估报告生成的鲁棒性。封装StructuredOutputInvoker统一处理结构化输出解析失败的重试逻辑，提升LLM调用的可靠性。
实现基于PostgreSQL + pgvector的RAG检索增强系统，采用文档分块（TokenTextSplitter）+ 阿里云text-embedding-v3向量化（1024维）+ HNSW索引存储，满足中小规模知识库的检索性能需求。
优化RAG检索效果，通过前置过滤（filterExpression）将知识库隔离逻辑下沉至向量库层执行， 配合Query Rewrite +动态参数矩阵+短Token物理命中判定，有效拦截RAG幻觉输出。
构建基于S3协议的统一存储抽象层，使用AWS S3 SDK兼容RustFS对象存储，实现面试附件的统一管理与持久化。
封装Redis + Lua分布式限流组件，实现滑动窗口算法支持GLOBAL/IP/USER多维度令牌桶限流，通过Hash Tag优化确保Redis Cluster下Key落在同一槽位，支持声明式配置fallback降级方法。
使用Apache Tika实现多格式文档解析，结合预编译正则清理控制字符、图片链接等噪声，针对 PDF禁用图片提取避免临时文件污染，针对DOCX禁用嵌入资源提取减少噪声。
基于iText 8实现PDF报告导出，内置珠圆玉润仿宋字体支持跨平台渲染，实现分数配色与多维度排版设计。
编写Docker Compose配置文件实现一键部署，通过环境变量管理敏感配置（如AI API Key）， 确保开发、测试与生产环境的一致性。
```
