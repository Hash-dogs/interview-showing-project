# 大模型微调
## 一、微调与预训练
* 预训练：将广泛输出与其对应的预计答案传给大模型（神经网络），提供不断微调大模型参数实现知识广泛学习（本质作用是教会知识）
* 微调：将特定任务输出与其对应的预计答案（如对话的场景示例，特定角色的对话模式等）传给大模型（神经网络），实现大模型对某一个功能的加强

## loRa(low rank adaption)方法降低微调是显存消耗
* 由于全量训练所需要的显存包含模型原参数，cache以及激活值、每个参数的更新梯度存储、优化器状态
* 采用loRa方法是将原大矩阵拆分为两个小矩阵相乘即a\*b -> (a\*r)\*(r\*b)，使得梯度与优化器状态需要存储的计算数据量大大减少。
* r值的取值认为指定，但是实验证明，r十分小也有很大的优化作用

## 二、多智能体：

### tool calling和路由型多智能体的区别是

| 维度 | Tool Calling | 路由型多智能体 |
|------|-------------|--------------|
| 执行主体 | 单一LLM + 工具集 | 多个独立Agent |
| 状态管理 | 单一上下文窗口 | 各Agent可有独立状态/记忆 |
| 并发能力 | 通常串行 | 可真正并行执行 |
| 复杂度 | 低 | 高 |
| 适用场景 | 单轮/短流程 | 多轮/长流程/复杂协作 |


路由型多智能体适合与多轮对话，并行执行以及专家agent需要隔离历史记忆的情况
