# 工具类

## 一、BeanOutputConverter（springAI自带工具类）
* 将大模型的json输出反序列化到对应的DTO类，返回DTO类对象
* 底层使用了jackson工具
* 能自动根据DTO生成响应的prompt的json输出格式
```
BeanOutputConverter<InterviewReport> converter = new BeanOutputConverter<>(InterviewReport.class);
// "返回一个JSON对象，包含以下字段（字段是根据InterviewReport生成）：summary(String), score(Integer), suggestions(List<String>)“
```

## 二、StructuredOutputInvoker（自定义工具类：统一封装结构化输出调用与重试策略。）
*  主要方法invoke包含调用大模型逻辑，大模型输出处理以及重试逻辑

## 三、PromptTemplate（映射.st文件进行prompt注入）
* 在构造函数中对其进行配置，将同一个包下的prompt的st文件地址映射到对应的对象中
* 在需要使用prompt的时候使用render（）函数即可，会将文件返回为String类型
* 注意.st文件最大的优势是可以使用占位符，通过render传参确定占位符的值：
  * render不传参说明对于的.st文件是不包含占位符的
  * render传参是一个map，key值是占位符名称，value是替换值
  * 如果出现有占位符不传参或者传参不完整不匹配会直接报错

## 四、LlmProviderRegistry（构建不同功能的ChatClient）
* createChatClient(id) 普通客户端：有 SkillsTool + Memory Advisor（有记忆）
* createVoiceChatClient(id) Voice客户端：有SkillsTool，但手动管理历史，不加Memory Advisor 模拟口头面试可用
* createPlainChatClient(id) Plain客户端：没有SkillsTool，没有Memory（用于出题/评分，要纯JSON）模拟面试出题可使用


## 五、异步线程操作CompletableFuture类（结合questionExecutor异步线程池）
1. 创建：
```
CompletableFuture<返回类型> future = CompletableFuture.supplyAsync(() -> {
    //lambda表达式
});
```

2. 获取结果
```
// 阻塞等待结果，会抛出受检异常，需要try-catch
String result = future.get();

// 阻塞等待，设置超时时间
String result = future.get(3, TimeUnit.SECONDS);

// 阻塞等待，不抛受检异常（推荐）
String result = future.join();

// 立即获取，没完成则返回默认值
String result = future.getNow("默认值");
```

3. 完成后回调操作
```
// thenApply：有返回值，对结果做转换
CompletableFuture<Integer> future = CompletableFuture
    .supplyAsync(() -> "hello")
    .thenApply(s -> s.length());  // lambda中的参数就是异步操作的返回值，"hello" → 5

// thenAccept：无返回值，消费结果
CompletableFuture<Void> future = CompletableFuture
    .supplyAsync(() -> "hello")
    .thenAccept(s -> System.out.println(s));

// thenRun：无返回值，不关心结果，只是任务完成后执行
CompletableFuture<Void> future = CompletableFuture
    .supplyAsync(() -> "hello")
    .thenRun(() -> System.out.println("任务完成了"));
```
